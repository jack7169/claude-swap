## Remaining implementation (Feature 1 + Feature 2 from the spec)

Follow TDD (red → green). Run tests with: `uv run --extra menubar pytest -q`
(689 passed / 3 skipped at handoff). Tests must NOT spawn rumps or osascript.

### Feature 1 — flat indented status view (`menubar.py`)
New pure helpers (unit-test each):
- `account_detail_lines(usage)` → per-window rows like
  `"5h:  5%   resets 18:59   in 4h 46m"`. Reuse `oauth.format_reset(resets_at)`
  → `(countdown, clock)` (same formatter the CLI uses). Mock `format_reset` in
  tests to avoid tz/now coupling. Omit a window with unknown `pct`; omit the
  reset segment when `resets_at` missing/unparseable.
- `group_running_instances(sessions, ides)` → `[(label, folder, sess_count,
  has_ide)]`, grouping identical to `switcher.status` (`switcher.py:1590-1614`),
  reusing `printer.entrypoint_label / ide_short_name / abbreviate_path`.
- `format_instance_row(group)` → `"VS Code   ~/Dev/TL-Starnav  (2 sessions, IDE)"`.

Wiring:
- `_snapshot()` (module fn ~`menubar.py:421`) adds `"instances"` from
  `process_detection.get_running_instances()`, wrapped so failure → `[]`.
- `rebuild_menu()` (~`menubar.py:607`): for each account emit the existing
  clickable name row (checkmark when active; `_make_switch_to(num)` callback —
  unchanged) THEN two disabled (callback=None) detail rows; after the accounts,
  a separator + a "Running instances" disabled header + one disabled row per
  group (skip the section entirely when no instances).

### Feature 2 — unified swap notifier
- New module `notify.py`: `notify(title, message)` posts a macOS notification via
  `osascript`. No-op when `sys.platform != "darwin"`; never raises; `timeout=5`.
  Pure `_build_notification_script(title, message)` handling AppleScript escaping
  (`\` and `"`) — unit-test this.
- `switcher.py`: add `self._on_switch = None` in `__init__` and a
  `set_switch_notifier(callback)` method. Add `_announce_switch(account_num,
  email)` that calls `self._on_switch(int(num), email)` if set (guard with
  try/except → `self._logger.debug`). Call `self._announce_switch(target_account,
  target_email)` before BOTH success returns in **`_perform_switch`**
  (`switcher.py:2185`; success returns ~line 2348 (activation) and ~2467
  (normal)). `_perform_switch` is the single chokepoint for `switch()` AND
  `switch_to()`. Put the calls OUTSIDE the `if emit_output:` blocks so JSON-mode
  and menu/auto swaps also notify.
- `cli.py` `main()`: right after `switcher = ClaudeAccountSwitcher(...)` and the
  root check, on macOS call `switcher.set_switch_notifier(lambda num, email:
  notify("claude-swap", f"Switched to Account-{num} ({email}) — restart Claude
  Code to apply (active within ~30s)."))`. Same instance is passed to
  `--menubar`, so CLI + menu + auto-switch all route through it.
- Remove the now-redundant `rumps.notification` swap messages in `menubar.py`:
  the body of `_notify_switched` and the `_notify_autoswitch` call (and method)
  — otherwise menu/auto swaps double-notify. KEEP the non-swap alerts
  ("Claude limit — no fresh account", "Auto-switch failed").
